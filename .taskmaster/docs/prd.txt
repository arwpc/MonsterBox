# Jaw Animation System - Product Requirements Document

## Project Overview

Create a jaw animation system for the MonsterBox application by implementing ChatterPi-inspired audio-servo synchronization functionality. This system will analyze audio volume levels in real-time and control a servo motor to animate a jaw that opens and closes in sync with speech/audio playback for any animatronic character (bears, skeletons, fish, demons, etc.).

## Business Objectives

- Enhance MonsterBox animatronic capabilities with realistic jaw animation for any character type
- Provide real-time audio-servo synchronization without disrupting existing audio systems
- Create a reusable framework for audio-driven servo animations across all animatronic types
- Maintain compatibility with existing MonsterBox character and scene systems

## Technical Requirements

### Core Functionality
1. **Audio Integration**: Monitor audio streams from currently selected character without blocking concurrent audio playback
2. **Servo Control**: Control servo motors using existing MonsterBox servo infrastructure (scripts/servo_control.py and routes/servoRoutes.js)
3. **Real-time Synchronization**: Analyze audio volume levels and translate to servo positions for realistic jaw movement
4. **Non-blocking Operation**: Work alongside existing MonsterBox functionality without interference
### System Architecture
1. **Audio Analysis Module**: Real-time volume detection and processing
2. **Servo Mapping Engine**: Volume-to-servo-position algorithms with configurable sensitivity
3. **Integration Layer**: Connect with existing MonsterBox audio and servo systems
4. **Configuration System**: Per-character and per-servo-type settings

### Performance Requirements
- Real-time audio processing with minimal latency (<50ms)
- Smooth servo movement without jitter
- CPU usage <10% additional overhead
- Memory usage <50MB additional

### Compatibility Requirements
- Work with existing soundController.js and sound_player.py
- Use existing servo control infrastructure
- Maintain character selection and scene playback functionality
- Support all current servo types in servos.json

## Technical Specifications

### Audio Processing
- Sample rate: 44.1kHz or 48kHz
- Bit depth: 16-bit minimum
- Analysis window: 10-50ms configurable
- Volume detection: RMS and peak analysis
- Frequency filtering: Optional bandpass for speech frequencies

### Servo Control
- Position range: 0-180 degrees configurable
- Update frequency: 20-50Hz
- Smoothing: Configurable interpolation
- Calibration: Per-servo min/max positions

### Configuration Options
- Audio sensitivity levels
- Servo response curves
- Smoothing parameters
- Frequency filters
- Character-specific presets

## User Stories

### Primary Users: Animatronic Operators
1. As an operator, I want to assign a jaw servo to any character so that it can have talking animation
2. As an operator, I want to test jaw movement with different audio files for bears, skeletons, fish, etc.
3. As an operator, I want to adjust sensitivity so jaw movement matches audio intensity for different character types
4. As an operator, I want to use jaw animation during scene playback for any animatronic

### Secondary Users: System Administrators
1. As an admin, I want to configure servo types and their parameters for different animatronic designs
2. As an admin, I want to monitor system performance during jaw animation across multiple characters
3. As an admin, I want to troubleshoot audio-servo synchronization issues for any character type

## Success Criteria

### Functional Success
- Jaw servo moves in real-time sync with audio playback for any character type
- System works with all existing MonsterBox audio features
- Configuration persists across sessions
- Multiple characters can have jaw animation capability simultaneously

### Performance Success

- Must use existing MonsterBox audio infrastructure
