# MonsterBox Animatronic Control System - Comprehensive Product Requirements Document

## Executive Summary

MonsterBox is a sophisticated animatronic control system built with Node.js/Express, designed for managing distributed Raspberry Pi-based animatronic systems in haunted attraction environments. This comprehensive PRD covers the complete system including infrastructure, jaw animation, SSH configuration, security, monitoring, testing, and deployment.

## Project Overview

The MonsterBox system encompasses multiple integrated components:

1. **Core Infrastructure**: Node.js/Express backend with comprehensive testing and monitoring
2. **Jaw Animation System**: ChatterPi-inspired audio-servo synchronization for realistic talking animatronics
3. **SSH-First Development**: Reliable remote development environment with PowerShell fallback prevention
4. **Security & Authentication**: JWT-based secure access with role-based permissions
5. **Monitoring & Logging**: Comprehensive log collection and real-time monitoring dashboard
6. **Hardware Integration**: GPIO, I2C, servo control, and sensor management for Raspberry Pi systems
7. **Scene Management**: Timeline-based scene orchestration and playback
8. **Backup & Recovery**: Automated backup systems and disaster recovery procedures

## SSH-First Development Environment Configuration

### Project Overview
Configure VS Code and Augment to ALWAYS use SSH for code deployment and execution, preventing any PowerShell fallback. The core problem is that Augment keeps failing SSH connections and silently reverting to local PowerShell execution on Windows instead of the intended remote Linux environment.

### Problem Statement
The current development workflow suffers from critical reliability issues:
- Augment agents frequently lose SSH connections to remote Raspberry Pi hosts
- When SSH fails, the system silently falls back to local Windows PowerShell execution
- Code intended for Linux deployment runs on Windows instead, causing deployment failures
- No monitoring or alerting when SSH connections fail
- Inconsistent remote platform detection leads to OS misidentification
- VS Code Remote-SSH occasionally reverts to local terminal execution

### Target Hosts
- skulltalker: 192.168.8.130 (user: remote, password: klrklr89!)
- coffin: 192.168.8.140 (user: remote, password: klrklr89!)
- orlok: 192.168.8.120 (user: remote, password: klrklr89!)

### Critical SSH Requirements

#### 1. SSH Connection Reliability
- Implement robust SSH connection management with automatic retry logic
- Configure SSH keepalive settings to prevent connection timeouts
- Add connection health monitoring and automatic reconnection
- Implement exponential backoff for failed connection attempts
- Create SSH connection pooling for multiple concurrent operations

#### 2. PowerShell Fallback Prevention
- Force VS Code Remote-SSH to NEVER fall back to PowerShell
- Implement safeguards to detect and prevent local PowerShell execution
- Add validation checks before any code execution to ensure remote context
- Create monitoring system to alert when SSH connection fails
- Implement hard stops when remote connection is unavailable

#### 3. Remote Platform Detection
- Set explicit remote platform detection to prevent OS misidentification
- Configure robust SSH settings with proper authentication and timeouts
- Implement platform verification before code execution
- Add OS detection validation in all deployment scripts
- Create fallback detection mechanisms for edge cases

#### 4. VS Code Integration
- Configure VS Code terminal integration to always use remote bash
- Ensure Augment Remote Agents maintain SSH connections throughout workflow
- Implement session persistence for long-running development tasks
- Configure automatic remote workspace selection
- Add remote context validation for all operations

#### 5. Monitoring and Verification
- Create verification system to test that all code execution happens remotely
- Implement real-time SSH connection status monitoring
- Add logging for all SSH connection attempts and failures
- Create dashboard for monitoring remote host connectivity
- Implement automated testing of SSH connection reliability
