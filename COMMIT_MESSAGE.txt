Phase 1: Orlok Bring-Up - Per-Character TTS and Deployment System

Implements complete per-character TTS configuration system with file-first
ElevenLabs API key management and automated deployment tools for Orlok and
all animatronics.

## Core Changes

### ElevenLabsConfigService - File-First Key Reading
- Modified getElevenLabsConfig() to prefer /etc/monsterbox/elevenlabs.key
- Falls back to ELEVENLABS_API_KEY environment variable if file not found
- Ensures consistent API key management across all animatronics
- File: services/elevenLabsConfigService.js

### Per-Character TTS Configuration
- Added getTTSConfigForCharacter(characterId) function
- Loads TTS config from data/character-{id}/ai-config/tts-config.json
- Falls back to global config if character-specific config doesn't exist
- Merges with defaults to ensure all required fields exist
- File: services/aiConfigStore.js

### Generate-and-Play Endpoint Update
- Updated /api/elevenlabs/generate-and-play to use per-character TTS config
- Removed hardcoded voice_id and TODO comment
- Returns voiceId in response for verification
- File: routes/api/elevenLabsApiRoutes.js

### Character TTS Config Fix
- Fixed character 4 (Skulltalker) empty voice_id
- All characters now have valid TTS configuration
- File: data/character-4/ai-config/tts-config.json

## Deployment Tools

### Deployment Script
- Automated deployment to any animatronic
- Syncs code while preserving character-specific data
- Checks ElevenLabs key file and permissions
- Installs dependencies if needed
- Restarts MonsterBox service and verifies status
- File: scripts/deploy-to-animatronic.sh

### Comprehensive Test Script
- 7-test suite for on-device validation
- Tests: key file, API validity, service status, PipeWire sinks,
  TTS config, speaker parts, TTS playback, logs
- File: scripts/orlok-bringup-test.sh

### Multi-Animatronic Test Script
- Tests all 4 animatronics sequentially
- Checks network, service, and TTS for each
- Reports success/failure summary
- File: scripts/test-all-animatronics.sh
