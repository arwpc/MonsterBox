# MonsterBox

MonsterBox 4.0 - Single-Node Animatronic Control System

## Overview

MonsterBox 4.0 is a comprehensive single-node animatronic control system designed to run on a Raspberry Pi 4B. It provides a unified platform for controlling and coordinating various hardware components with built-in calibration, safety features, and real-time testing capabilities.

## Platform

- **Hardware**: Raspberry Pi 4B
- **Architecture**: Single-node control system

## Key Features

### Hardware Support
MonsterBox 4.0 unifies control for multiple hardware components:
- **Servos**: Precise servo motor control with verified movement
- **Actuators**: Support for various actuator types
- **Lights**: Lighting control and effects
- **Audio**: Full audio library integration
- **Sensors**: Sensor input processing
- **Webcam**: Camera input for vision features

### Core Capabilities
- **Reusable Poses**: Define and recall animatronic poses for consistent performance
- **Full Audio Library**: Complete audio support with PipeWire integration for reliable audio routing
- **OpenCV Motion Detection**: Real-time motion detection capabilities
- **Head Tracking**: Advanced head tracking using computer vision
- **Scene Queuing**: Queue and manage multiple scenes for complex performances
- **Calibration Tools**: Built-in calibration utilities for hardware setup
- **Safety Features**: Comprehensive safety mechanisms to protect hardware
- **Real-Time Testing**: Live testing capabilities for immediate feedback

## Recent Improvements

MonsterBox 4.0 includes several important fixes and enhancements:
- **Stable APIs**: Refined and stabilized control APIs
- **Audio Routing**: Improved audio routing through PipeWire
- **Safer Defaults**: Enhanced default configurations for improved safety
- **Verified Servo Movement**: Thoroughly tested and verified servo control
