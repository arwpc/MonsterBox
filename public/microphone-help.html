🎤 Microphone Access Help

⚠️ Microphone Access Denied

The microphone component cannot access your microphone because this page is not served over a secure connection (HTTPS).

📋 Why This Happens

Modern web browsers require a secure context (HTTPS) to access sensitive hardware like microphones and cameras. This is a security feature to protect users from malicious websites.

Current connection: HTTP

Required connection: HTTPS

🔧 Solutions

Option 1: Use HTTPS (Recommended)

Access MonsterBox using https:// instead of http://
Example: https://skulltalker:3000 instead of http://skulltalker:3000
You may need to accept a security certificate warning

Option 2: Use Localhost

Access MonsterBox using https://localhost:3000
Or use https://127.0.0.1:3000

Option 3: Configure SSL Certificate

Set up a proper SSL certificate for your MonsterBox server
This provides the most secure and seamless experience

🔍 Browser Compatibility

The microphone features require:

Chrome 47+ or Firefox 36+ or Safari 11+
HTTPS connection (or localhost)
User permission to access microphone
