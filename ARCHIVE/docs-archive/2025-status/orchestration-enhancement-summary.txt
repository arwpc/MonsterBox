MonsterBox Orchestration Page Enhancements
==========================================
Date: October 25, 2025
Commit: 32a53563

FEATURES ADDED
==============

1. Webcam Streaming Preview
   - Each online animatronic card now displays live webcam feed
   - Stream URL: http://<animatronic-ip>:3000/setup/webcam/api/parts/auto/stream
   - 150px height preview with object-fit: cover
   - Falls back gracefully if webcam not available (hides image on error)
   - Black background placeholder for consistent layout

2. AI Interaction Controls
   - Text input field for each online animatronic
   - Two action buttons per animatronic:
     * "Ask AI" - Sends question to animatronic's AI agent via /conversation/api/ask-ai
     * "Say" - Makes animatronic speak text via /conversation/api/say
   - Compact button group layout (50/50 split)
   - Real-time feedback in command log
   - Input clears automatically after successful action

3. Enhanced Animatronic Cards
   - Compact card design with better visual hierarchy
   - Character name display (fetched from each animatronic)
   - Clickable header to open animatronic web interface in new tab
   - Status indicator (online/offline) with pulsing animation
   - IP:port display at bottom
   - Responsive grid: 4 columns on XL, 3 on LG, 2 on MD, 1 on SM

4. Improved Service Integration
   - Enhanced healthCheck() to fetch character information
   - Added characterName to status response
   - Graceful fallback if character data unavailable
   - Maintains existing functionality while adding new features

USAGE
=====

Ask AI Feature:
1. Type question in text input (e.g., "What's your favorite scary movie?")
2. Click "Ask AI" button
3. AI processes question through character's personality
4. Response logged in command log
5. Animatronic may speak response (if configured)

Make Say Feature:
1. Type text in input (e.g., "Happy Halloween!")
2. Click "Say" button
3. Animatronic speaks text immediately
4. Success/failure logged in command log

Webcam Streaming:
- Automatically loads when animatronic is online
- Updates in real-time (MJPEG stream)
- Falls back silently if webcam not configured
- No user interaction required

COMPATIBILITY
=============

Browser Requirements:
- Modern browsers with MJPEG support (Chrome, Firefox, Safari)
- JavaScript enabled
- CSS Grid support

Network Requirements:
- All animatronics must be on same network (192.168.8.x)
- Port 3000 accessible on each animatronic
- CORS not required (same origin or permissive CORS)

Fallback Behavior:
- Webcam fails: Image hidden, card remains functional
- AI endpoint fails: Error logged, other features work
- Character info unavailable: Card shows without name
- Offline animatronic: Shows offline state, no interactive elements
